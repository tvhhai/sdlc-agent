# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A build engine that turns canonical agent definitions (`agents/*.yaml`) into AI-tool-specific formats. One source of truth, three render targets: universal (`AGENTS.md` + `.sdlc/agents/`), Claude Code (`.claude/agents/`), and GitHub Copilot (`.github/copilot-instructions.md` + `.github/prompts/`).

## Commands

There is no compile step — `tsx` runs TypeScript directly.

```bash
pnpm install
pnpm sdlc build       # render all targets + write .sdlc/build-manifest.json
pnpm sdlc validate    # schema + referential integrity + target names
pnpm sdlc init        # scaffold sdlc.config.yaml

pnpm test             # vitest run (all packages)
pnpm test packages/core/src/__tests__/merger.test.ts   # single test file
pnpm vitest run -t "deep-merges variables"             # single test by name
pnpm typecheck        # tsc --noEmit (pnpm build is an alias for this)
pnpm lint             # biome check .
```

## Critical rule: generated files

`AGENTS.md`, `.sdlc/`, `.claude/agents/`, `.github/copilot-instructions.md`, and `.github/prompts/` are **build outputs**, committed on purpose. Never edit them by hand — edit `agents/*.yaml`, `templates/`, `policies/`, or the adapters, then run `pnpm sdlc build`. The build compares file hashes against `.sdlc/build-manifest.json` and warns on drift; integration tests in `packages/cli/src/__tests__/real-agents.test.ts` fail if rendered output diverges from the committed files, so **always rebuild and commit the regenerated outputs together with source changes**.

## Architecture

Pipeline (all orchestrated by `packages/cli/src/commands/build.ts`):

```
sdlc.config.yaml ─► loadConfig ─► mergeConfigs (deep-merge, last wins)
agents/*.yaml    ─► loadAgents (yaml parse) ─► Zod schema (packages/core/src/schema.ts)
                 ─► resolveAgent ({{variable}} interpolation from config.variables)
                 ─► each adapter.render(agents, ctx) ─► OutputFile[] ─► engine writes files + manifest
```

- **`packages/core/`** — Zod schema, loader, variable resolver, config merger, shared types (`AgentDef`, `BuildContext`, `OutputFile`, `ToolAdapter`). The schema is strict: kebab-case ids, semver versions, phase enum (`requirement|planning|architecture|coding|testing|review`), non-empty workflow.
- **`packages/cli/`** — `sdlc init|validate|build`. `validate` also checks references: every `output_template` must exist on disk, every `policies` entry needs `policies/<name>.md`. `extends`/`imports` are accepted by the schema but **not implemented** — validation fails loudly on them (Phase 2 feature).
- **`packages/adapters/{universal,claude-code,copilot}/`** — each exports a `ToolAdapter`. New adapters must be registered in the hardcoded `ADAPTERS` map in `build.ts`.

### Adapter contract (docs/adapter-contract/ADAPTER_CONTRACT.md)

- `render()` must be **pure and deterministic** — no timestamps, no I/O, no randomness. This is what makes snapshot tests and drift detection work.
- Adapters never write files; the build engine owns all filesystem writes.
- Every output file carries the string `Generated by sdlc-agents`.
- Respect `model_variants.<tool>` for your tool only, ignore other tools' variants.
- Each adapter package needs path contract, content contract, and snapshot tests under `src/__tests__/`.

### Adapter-specific mappings

- **claude-code**: `model_hint` → model id (`fast`→Haiku, `balanced`→Sonnet, `high-reasoning`→Opus, unknown→Sonnet); description gets `[SDLC:<phase>]` prefix (drives Claude Code auto-routing).
- **copilot**: prompt files use `mode: agent` frontmatter; frontmatter description is the first line only, JSON-quoted.

## docs/

MD files are the source of truth; each has a collocated HTML viewer that must be kept in sync. Full rules (layout structure, TOC ids, how to add a doc) live in [docs/CLAUDE.md](docs/CLAUDE.md) — read it before touching anything under `docs/`.

Key docs: `docs/sa-design/` (full SA design + roadmap), `docs/adapter-contract/` (adapter rules), `docs/codebase-guide/` (per-file map of the repo).

## Conventions

- Biome for lint/format (tab indentation, double quotes). Run `pnpm lint` before committing.
- `spike/` is the Phase 0 prototype, kept for reference — don't extend it.
- Vitest snapshot tests cover adapter output; if you intentionally change rendered format, update snapshots **and** rebuild the committed outputs.
