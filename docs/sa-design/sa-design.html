Agentic SDLC Agents Set

Solution Architecture Document — bộ AI agent definitions tái sử dụng, tool-agnostic, phủ toàn bộ vòng đời phát triển phần mềm từ Requirement → Maintain.

1. Executive Summary

Agentic SDLC Agents Set là một thư viện các AI Agent được đặc tả sẵn (agent definitions, prompts, workflows, skills) cho từng giai đoạn của vòng đời phát triển phần mềm (SDLC), được đóng gói theo cách tool-agnostic: cùng một agent có thể chạy trên mọi AI agent/IDE agent hiện nay — Claude Code, GitHub Copilot, OpenAI Codex, Cursor, Windsurf, Gemini CLI, Zed, Cline… — mà không phải viết lại.

Cơ chế: ưu tiên 1 chuẩn mở universal (AGENTS.md) mà mọi tool đều đọc được, cộng thêm native adapter cho tool nào có format riêng mạnh hơn.

Team mới chỉ cần install bộ agents là có ngay quy trình SDLC chuẩn có AI hỗ trợ — không phải tự viết prompt/agent từ đầu. Cùng coding convention, cùng template tài liệu, cùng quy trình review giữa các team. Một nguồn maintain duy nhất → build ra nhiều format cho từng tool. Giống Terraform modules nhưng cho AI agents.

2. Problem Statement

2.1 Hiện trạng

  • Mỗi developer/team tự viết prompt riêng → chất lượng không đồng đều, trùng lặp công sức.
  • Mỗi AI tool có format cấu hình riêng: Claude Code (CLAUDE.md, .claude/agents/), Copilot (.github/prompts/), Codex (AGENTS.md), Cursor (.cursorrules), Gemini (GEMINI.md)…
  • Kiến thức nghiệp vụ (coding standard, review checklist, tài liệu) không được nhúng nhất quán vào AI workflow.
  • Không có versioning, testing, phân phối cho prompt/agent.

2.2 Bài toán cần giải

Làm sao để viết agent một lần, dùng được trên mọi AI tool, phủ đủ các phase SDLC, và phân phối/cập nhật dễ dàng cho nhiều team?

6.2 Adapter Strategy — 2 tầng

Dựa trên chuẩn mở AGENTS.md — chuẩn de-facto được Codex, Cursor, Windsurf, Gemini CLI, Zed, Amp, Roo, Cline, Jules, Devin… cùng hỗ trợ. Chỉ là Markdown thuần + đường dẫn tương đối — mọi AI agent có thể đọc file đều dùng được. Chỉ viết khi tool có cơ chế riêng đáng giá (subagents, slash commands, rule scoping…). Tool mới ra → tự động được hỗ trợ ở Tầng 0, không cần code thêm.

Bảng adapters

AdapterOutput nativeLợi ích thêm
claude-codeCLAUDE.md, .claude/agents/*.md, skills, commands, hooksSubagent thật (context riêng), slash commands, auto-trigger skills, hooks
copilot.github/copilot-instructions.md, .github/prompts/*.prompt.mdPrompt files gọi nhanh, Copilot coding agent trên GitHub.com
cursor.cursor/rules/*.mdcRule scoping theo glob, auto-attach theo file đang mở
windsurf.windsurf/rules/*.md, workflowsCascade workflows, rule activation modes
geminiGEMINI.mdTích hợp Gemini CLI native
codex / antigravity / zed / ampAGENTS.md (dùng chung Universal)Không cần adapter riêng — chi phí hỗ trợ = 0

6.3 Developer Journey

1. npx @org/sdlc-agents init

2. Commit generated files
Files được generate vào repo (.claude/, .github/, …) → commit vào git như code bình thường
3. Dev dùng agent trong tool quen thuộc
Claude Code: /plan "thêm tính năng X" → planner agent
Copilot: @workspace /review-pr → code-reviewer prompt
4. Org cập nhật bộ agents
npx @org/sdlc-agents update → re-build → PR tự động

7. Customization & Extensibility Model

"Convention over configuration, but everything is overridable." — zero-config cho người mới, override từng dòng prompt cho người nâng cao, không bao giờ phải fork repo gốc.

CấpTênNgười dùng làm gìEffort
L1ConfigureBật/tắt agents, set variables (stack, language, team) trong config5 phút
L2OverrideThay thế từng phần: policies, templates, checklist, từng step trong workflow30 phút
L3ExtendViết agent mới của riêng dự án, kế thừa (extends) agent có sẵn1–2 giờ
L4Plug-inViết adapter cho tool mới, hook vào build pipeline, publish preset cho team khác dùng1–2 ngày
 
# sdlc-agents.config.yaml
targets: [universal, claude-code, copilot]   # universal phủ mọi tool còn lại
variables:
  language: vi
  stack: nestjs-postgres
  team: payment-squad
agents:
  enable: [planner, coder, code-reviewer]
  disable: [ux-reviewer]

Override chỉ chứa phần khác biệt → khi base cải tiến prompt, user hưởng cải tiến mà vẫn giữ tùy chỉnh riêng. Ba toán tử: replace, append/prepend, insert_after/insert_before.

# overrides/code-reviewer.yaml
agent: code-reviewer
patch:
  workflow:
    insert_after: "Chạy checklist security"
    step: "Kiểm tra naming theo quy ước payment-squad"
  output_template: ./our-review-template.md
  prompt_append: |
    Luôn kiểm tra idempotency cho mọi API thanh toán.
Tình huốngLối thoát
DSL không biểu diễn đượctool_specific: block — viết raw content cho riêng 1 tool
Muốn sửa trực tiếp file outputeject mode: copy file generated thành file thường, managed: false
File generated bị sửa tayBuild engine cảnh báo drift — SHA-256 manifest
Cần quay về mặc địnhnpx sdlc-agents reset <agent>

11. Roadmap & Milestones

Phase 0 — Foundation (Tuần 1–2) ✅
Chốt DSL schema, setup monorepo, Zod schema + validator, spike: render 2 agents sang Claude Code + Universal — output valid ✅
Phase 1 — MVP (Tuần 3–6) ✅ tech complete
Build engine + 3 adapters (universal, claude-code, copilot), CLI init/build/validate, 6 agents MVP, CI xanh, README, drift detection, adapter contract doc. Còn lại: dogfood + pilot team.
Phase 2 — Hardening (Tuần 7–10) ⬜
Multi-layer override, patch-based override, extends agent, escape hatch, CLI update + SemVer, snapshot test toàn bộ output, eval harness cơ bản, thêm agents: test-strategist, security-auditor, release-manager.
Phase 3 — Scale (Quý sau) ⬜
Native adapters đợt 2 (Cursor, Windsurf, Gemini CLI, Cline/Roo), compatibility matrix, docs site + catalog, telemetry opt-in, marketplace/registry nội bộ.

13. Risks & Mitigations

R1
AI tool thay đổi format config liên tục
Mitigation: Adapter pattern cô lập thay đổi; contract test theo từng tool version; theo dõi changelog
Cao Cao
R2
Lowest common denominator: feature mạnh của 1 tool không map được sang tool khác
Mitigation: DSL có section tool_specific cho phép khai báo phần riêng từng tool
Cao Trung bình
R3
Chất lượng output agent không ổn định giữa các model
Mitigation: Eval harness + golden test cases; model_hint trong DSL
Trung bình Cao
R4
Team không adopt vì "thêm một thứ phải học"
Mitigation: init 1 lệnh, zero-config; dogfooding + champion tại mỗi team
Trung bình Cao
R5
Prompt chứa thông tin nội bộ bị leak khi commit public
Mitigation: Lint rule + .gitignore guidance + phân tách policies internal
Thấp Cao
R7
Supply-chain prompt injection: skill import từ cộng đồng có thể chứa chỉ dẫn độc hại ẩn
Mitigation: Human review bắt buộc khi vendor lần đầu + khi bump pin; lint scan pattern đáng ngờ; pin theo commit hash
Thấp Rất cao

14. Open Questions

Q1 — Ngôn ngữ prompt: ✅ CHỐT: Prompt viết tiếng Anh; output template theo config language (default en).
Q2 — Tool ưu tiên thứ 2: ✅ CHỐT: GitHub Copilot — native adapter Phase 1 là Claude Code + Copilot.
Q3 — OSS vs Private: Đề xuất Open Core — engine + Universal + catalog cơ bản OSS; hosted policy management + eval harness + SLA commercial. Quyết định trước Phase 1.
Q6 — Trigger trên tầng Universal: Claude Code có auto-trigger; tool chỉ đọc AGENTS.md dùng model tự đọc chỉ mục hoặc quy ước lệnh tay. Đây là ceiling chấp nhận được của tầng Universal.
Q4 — Owner policies chuẩn org: Cần 1 owner (Engineering Excellence group) duyệt thay đổi checklist/convention. Chưa chốt.
Q7 — Ngân sách eval: Eval harness gọi API model thật → quota, chi phí, phân bổ. Cần chốt trước Phase 2.
