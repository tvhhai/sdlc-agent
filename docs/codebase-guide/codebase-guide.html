Codebase Folder Guide

Hướng dẫn đầy đủ về cấu trúc repo sdlc-agent — từng folder, từng package, và đánh giá đối chiếu SA design với code thực tế (mục 21).

1. Tổng quan kiến trúc

Mục tiêu: duy trì bộ định nghĩa AI agent theo mô hình "write once, render everywhere": agent được viết một lần ở dạng canonical YAML, sau đó build engine render ra các format nhiều AI tools có thể đọc được.

agents/*.yaml, templates/*.md, policies/*.md, sdlc.config.yaml packages/core/ (schema, loader, resolver, merger) + packages/cli/ (init, build, validate) packages/adapters/universal, claude-code, copilot

Build flow

sdlc.config.yaml ──► packages/cli runBuild() agents/*.yaml ──► packages/core loadAgents() ──► AgentSchema validation ──► resolveAgent(variables) templates/*.md ──► BuildContext policies/*.md ──► BuildContext │ ToolAdapter.render() │ │ │ ▼ ▼ ▼ AGENTS.md .claude/agents/ .github/prompts/ .sdlc/agents/ .github/copilot-instructions.md

2. Root Files

package.json Root monorepo. Scripts: pnpm sdlc (CLI via tsx), pnpm test (vitest), pnpm typecheck (tsc --noEmit), pnpm lint (biome). Runtime deps: yaml, zod.
pnpm-workspace.yaml Định nghĩa workspace packages: packages/* và packages/adapters/*. Nhờ file này các package depend nhau qua workspace:*.
tsconfig.json TypeScript strict mode, target ES2022, moduleResolution bundler. Alias @sdlc-agents/core → ./packages/core/src/index.ts. Include: packages/*/src/**, spike/**.
sdlc.config.yaml Config runtime của build engine. Khai báo targets (universal, claude-code, copilot) và variables (language: en). Nếu không có file, core dùng defaults.
biome.json Lint + format config. Excludes: docs, spike/output, .sdlc, .claude, .github/prompts, __snapshots__. Override: noNonNullAssertion off cho __tests__.
AGENTS.md Generated output của universal adapter. Entry point theo convention AGENTS.md. Không sửa tay — regenerate bằng pnpm sdlc build.
README.md Quickstart: install → validate → build → use agent. Bảng 6 agents, repo layout, configuration, validation rules, dev commands.

4. templates/ & policies/

templates/

Mẫu tài liệu mà agents dùng khi sinh output:

  • plan.md — implementation plan cho planner
  • review-report.md — review report cho code-reviewer
  • prd.md — PRD cho requirement-analyst
  • hld.md — HLD/ADR cho solution-architect
  • test-plan.md — test plan cho test-generator

Build engine load vào BuildContext.templates. Adapter hiện render reference đến template, chưa inline body. Đây là artifact hướng dẫn cho AI agent.

policies/

Guideline và checklist dùng bởi agent:

  • conventions.md — naming, function length, error handling, testing, TypeScript rules
  • security-checklist.md — input validation, auth/authz, secrets, dependency review, logging/PII

Tương tự templates — load vào BuildContext.policies. Universal adapter render link tương đối; Claude adapter render path string; Copilot chưa render policies section rõ ràng.

6. packages/core/

Library core @sdlc-agents/core — không phụ thuộc CLI UI. Cung cấp schema, loader, config, resolver, merger và shared types.

schema.ts Zod schema cho canonical agent. Components: ModelHint, WorkflowStep, ImportEntry, InputDef, ModelVariant, AgentSchema. Validation: id kebab-case, version semver, phase enum, description ≥ 10 chars, workflow ≥ 1 step, import license whitelist.
types.ts Shared interfaces: OutputFile (path + content), Diagnostic (severity + message), ResolvedConfig (targets, rootDir, variables, agentsDir), BuildContext (config + templates map + policies map), ToolAdapter (name + render + optional validate).
config.ts Đọc sdlc.config.yaml. Default: targets ["universal","claude-code"], variables {language:"en"}, agentsDir "agents". Parse với ConfigFileSchema rồi deep-merge lên default.
loader.ts Load agent YAML từ folder. Sort file deterministic → parse YAML → validate với AgentSchema.parse → trả AgentDef[]. Throw ZodError khi invalid.
resolver.ts resolveVariables(text, vars): thay {{key}} bằng vars[key], giữ nguyên nếu key không tồn tại. resolveAgent(agent, vars): serialize → resolve → parse lại. Không mutate original.
merger.ts mergeConfigs(base, ...overrides): scalar fields last-wins, deep-merge variables, không mutate base. Nền móng cho config multi-layer (chưa wire vào CLI).

7. packages/cli/

CLI package @sdlc-agents/cli. Entry: src/index.ts dùng commander.

commands/build.ts runBuild(cwd): load config → load agents → validate contracts → render via adapters → write files → write manifest.
Drift detection: SHA-256 hash mỗi file generated vào .sdlc/build-manifest.json; cảnh báo "Drift detected" nếu file bị sửa tay.
commands/validate.ts runValidate(agentsDir): load agents → validateProjectContracts() → log errors → trả boolean.
commands/project-validation.ts validateProjectContracts(agents, projectRoot): fail nếu duplicate id, missing output_template, missing policy file, agent dùng extends hoặc imports (unsupported — tránh silent no-op).
commands/init.ts Tạo sdlc.config.yaml mặc định nếu chưa có. Hiện scaffold đơn giản; interactive wizard (clack) là Phase 2.
Build ghi SHA-256 hash vào .sdlc/build-manifest.json. Lần build sau, file nào bị sửa tay sẽ được cảnh báo "Drift detected — N files were manually edited" rồi regenerate từ canonical sources.

8. packages/adapters/

Output: AGENTS.md + .sdlc/agents/<id>.md (1 file per agent)

renderIndex(): tạo AGENTS.md với heading, hướng dẫn kích hoạt, danh sách agent với link đến .sdlc/agents/.

renderAgent(): per-agent Markdown với phase/version/model, description, inputs, workflow, template ref, policy links, generated footer.

Footer chuẩn: "Generated by sdlc-agents. Universal tier — works on any AI tool. Run sdlc build to regenerate."

Output: .claude/agents/<id>.md (1 file per agent)

YAML frontmatter: name, description (prefixed [SDLC:<phase>] — drives auto-routing), model (fast→haiku, balanced→sonnet, high-reasoning→opus), tools.

Body: workflow, optional Claude-specific note từ model_variants.claude.prompt_append.

Output: .github/copilot-instructions.md + .github/prompts/<id>.prompt.md per agent

YAML frontmatter: mode: agent, description.

Body: title/phase, description, optional Copilot note từ model_variants.copilot.prompt_append, inputs, workflow, template ref, footer.

Note: Copilot adapter hiện không render policies section — chỉ thể hiện qua workflow steps nếu agent mention đến policy.

Required tests (mỗi adapter)

  • Path contract test — exact output paths
  • Content contract test — id, phase, workflow steps
  • Snapshot test — full rendered file

20. Mental Model ngắn gọn

Nơi định nghĩa agent. Sửa đây để thêm/đổi agent, workflow, tools. Đảm bảo agent hợp lệ (Zod schema) và resolve biến {{...}}. Điều phối build: đọc config → validate → render → write. Quyết định output file format cho từng AI tool. Kết quả generated — commit để consumer dùng ngay. Tài liệu tham chiếu để agent tạo output đúng format và đúng rule.

21. Đánh giá bổ sung sau khi review SA design và codebase 2026-06-11

Canonical agents ✅ · Build engine ✅ · 3 adapters ✅ · Validation chặt ✅ · CI xanh ✅ · Drift detection ✅ · README ✅ · Adapter contract doc ✅. Còn lại: dogfood + 1 pilot team để đạt exit criteria.
  • Hướng sản phẩm đúng pain point: copy prompt/agent config theo từng tool, khó version, khó test.
  • Kiến trúc canonical DSL → build engine → adapters đúng hướng. agents/*.yaml là source of truth, adapter chỉ lo đóng gói.
  • Universal-first tốt: AGENTS.md + .sdlc/agents/*.md tạo baseline portable cho Codex, Cursor, Windsurf, Gemini CLI, Zed...
  • Adapter pattern gọn và dễ mở rộng: mỗi adapter chỉ cần implement ToolAdapter.render().
  • Nền test tốt cho MVP: 53 tests, 9 files — schema, loader, resolver, CLI build/validate, adapter snapshot.
  • SA design đã nhìn đúng rủi ro dài hạn: prompt supply-chain, versioning, eval harness, policy layer, adapter compatibility.
  • Scope SA design khá rộng — registry, update mechanism, telemetry, eval harness, marketplace, skill import, multi-layer config đều có giá trị nhưng MVP quá sớm sẽ loãng delivery.
  • ✅ Đã xử lý silent no-op: imports/extends tồn tại trong schema nhưng validate/build hiện fail rõ ràng với message "not supported yet (planned: Phase 2)" — packages/cli/src/commands/project-validation.ts.
  • ✅ Đã có drift detection: SHA-256 manifest tại .sdlc/build-manifest.json — packages/cli/src/commands/build.ts.
  • Templates/policies hiện là reference cho AI agent, chưa inline/render body — chấp nhận được ở MVP.
  • Claim "support mọi AI tool": đúng hơn là universal tạo portable baseline; native UX vẫn cần adapter riêng và compatibility test.
  1. ✅ Adapter contract — docs/adapter-contract/ADAPTER_CONTRACT.md: interface, determinism bắt buộc, output paths, generated footer, required tests, quy trình thêm adapter mới.
  2. ✅ Reference validation — project-validation.ts: fail rõ nếu missing template/policy, duplicate id, unsupported extends/imports.
  3. ✅ Generated output strategy: commit generated files; footer marker Generated by sdlc-agents chuẩn hóa; drift detection qua manifest.
  4. ✅ MVP exit criteria: validate, build deterministic, adapter tests, CI xanh, README — hoàn tất. Còn: dogfood + pilot.

Dự án đang có nền tảng tốt và đang giải quyết đúng vấn đề: biến prompt/agent workflow thành artifact có schema, version, build và test. Giá trị lớn nhất không nằm ở từng prompt riêng lẻ, mà nằm ở việc chuẩn hóa cách team đóng gói và phân phối SDLC agents trên nhiều AI tools.

Hướng tiếp theo nên giữ MVP thật gọn: canonical agents, deterministic build, 3 adapters, validation chặt và CI đúng. Khi nền này chắc, các phần lớn hơn như registry, import community skills, eval harness và update mechanism sẽ dễ thêm hơn nhiều.
