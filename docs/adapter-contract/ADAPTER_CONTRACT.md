# Adapter Contract

Every adapter converts canonical agent definitions (`agents/*.yaml`) into the native
format of one AI tool. This document is the contract an adapter must satisfy to be
registered in the build engine. It exists so adapters stay consistent, testable, and
safe to regenerate.

## Interface

An adapter implements `ToolAdapter` from `@sdlc-agents/core`:

```typescript
interface ToolAdapter {
  readonly name: string;                                      // registry key, kebab-case
  render(agents: AgentDef[], ctx: BuildContext): OutputFile[];
  validate?(outputs: OutputFile[]): Diagnostic[];             // optional self-check
}
```

Rules:

- `render` must be **pure and deterministic**: same agents + same context → byte-identical
  output. No timestamps, no random ids, no network calls, no filesystem reads.
  (Determinism is what makes drift detection and snapshot testing possible.)
- `OutputFile.path` is **relative to the consumer project root** and must use `/` separators.
- An adapter must not write files itself — the build engine owns all filesystem writes
  and records each file's hash in `.sdlc/build-manifest.json` for drift detection.

## Required output properties

Every generated file must:

1. **Carry a generated footer** so humans (and drift tooling) know it is machine-owned:
   `_Generated by sdlc-agents. … run \`sdlc build\` to regenerate._` (exact wording may
   vary per adapter, but the string `Generated by sdlc-agents` must appear).
2. **Render every load-bearing field** of the agent definition: `description`, `inputs`
   (with required markers), `workflow` (ordered, with `ref` hints), `output_template`
   reference and `policies` references when present.
3. **Respect `model_variants`**: if the agent declares a variant for this adapter's tool
   (e.g. `model_variants.claude.prompt_append`), include it; ignore other tools' variants.

## Required tests

Each adapter package must contain, under `src/__tests__/`:

- **Path contract test** — asserts the exact output paths for a representative agent.
- **Content contract test** — asserts agent id, phase, and workflow steps appear in output.
- **Snapshot test** — at least one `toMatchSnapshot()` on a full rendered file, so any
  format change shows up in review.

## Registered adapters

| Adapter | `name` | Output paths | Notes |
|---------|--------|--------------|-------|
| Universal | `universal` | `AGENTS.md` (index) + `.sdlc/agents/<id>.md` (one per agent) | The baseline tier — plain Markdown readable by any AI tool (Codex, Cursor, Windsurf, Gemini CLI, Zed, …). Always keep this target enabled. No auto-routing: users invoke agents by name or by opening the file. |
| Claude Code | `claude-code` | `.claude/agents/<id>.md` (one per agent) | YAML frontmatter: `name`, `description` (prefixed `[SDLC:<phase>]` — drives auto-routing), `model` (mapped from `model_hint`: fast → Haiku, balanced → Sonnet, high-reasoning → Opus), `tools`. Applies `model_variants.claude`. |
| Copilot | `copilot` | `.github/copilot-instructions.md` (index) + `.github/prompts/<id>.prompt.md` (one per agent) | Prompt files use `mode: agent` frontmatter with a one-line `description`. Applies `model_variants.copilot`. |

## Adding a new adapter

1. Create `packages/adapters/<tool>/` with `src/index.ts` exporting a `ToolAdapter`.
2. Satisfy the output properties and tests above.
3. Register it in the `ADAPTERS` map in `packages/cli/src/commands/build.ts`.
4. Add the target name to `sdlc.config.yaml` docs and the README.
5. Add a row to the table above, including compatibility notes (which tool versions
   were verified, any format caveats).

> Phase 2 (per SA design FR-11) will replace the hardcoded `ADAPTERS` map with a plugin
> interface (`adapters: ["@org/sdlc-adapter-x"]` in config). Until then, adapters live
> in this monorepo.
