Quy ước Adapter (Adapter Contract)

Bộ quy ước bắt buộc mọi adapter phải đáp ứng để được đăng ký vào build engine. Xác định interface, quy tắc tính bất biến (determinism), thuộc tính đầu ra bắt buộc và yêu cầu kiểm thử.

Interface

Một adapter sẽ implement interface ToolAdapter từ package @sdlc-agents/core:

// packages/adapters/<tool>/src/index.ts
interface ToolAdapter {
  readonly name: string;                                      // registry key, dạng kebab-case
  render(agents: AgentDef[], ctx: BuildContext): OutputFile[];
  validate?(outputs: OutputFile[]): Diagnostic[];             // tự kiểm tra (optional)
}

Quy tắc cốt lõi

R1 Bất biến & thuần khiết (Pure & deterministic): cùng danh sách agents + cùng context ➔ kết quả đầu ra phải giống nhau đến từng byte. Không ghi timestamp, không sinh ID ngẫu nhiên, không gọi network, không đọc file ngoài luồng. Tính bất biến là điều kiện bắt buộc để chạy cơ chế phát hiện drift và kiểm thử snapshot.
R2 Đường dẫn tương đối: OutputFile.path là đường dẫn tương đối so với thư mục gốc của dự án người dùng (consumer project root) và bắt buộc phải sử dụng dấu phân tách /.
R3 Không tự ghi tệp tin: một adapter tuyệt đối không tự ghi file ra ổ đĩa — Build Engine sẽ chịu trách nhiệm ghi toàn bộ file và lưu mã hash của từng file vào .sdlc/build-manifest.json để phục vụ cơ chế phát hiện drift.

Render adapters vs host installers

Định hướng Hybrid thêm trải nghiệm installer/wizard qua npx sdlc-agents init, nhưng không đổi ranh giới trách nhiệm của adapter.

Là hàm thuần từ canonical agents + build context sang OutputFile[]. Adapter quyết định path và content cho một target format, nhưng không đọc host state và không tự ghi file. Hỏi hoặc detect AI host, chọn targets/presets/scope, ghi sdlc.config.yaml, rồi gọi build engine để render output lần đầu.
Logic installer không được nằm trong adapter.render(). Ngay cả target tương lai như Agent Skills (SKILL.md) cũng nên là render adapter deterministic; quyết định project-local hay user-global thuộc về wizard.

Thuộc tính đầu ra bắt buộc

Mỗi tệp tin được tạo ra bắt buộc phải đáp ứng cả 3 quy định sau:

  1. Chứa footer đánh dấu tự động: để lập trình viên (và các công cụ phát hiện drift) biết đây là file tự động tạo ra — chuỗi chữ Generated by sdlc-agents bắt buộc phải xuất hiện (cú pháp chính xác có thể khác nhau tùy từng adapter).
  2. Hiển thị đầy đủ các trường thông tin cốt lõi của agent: description, inputs (kèm ký hiệu bắt buộc), workflow (theo thứ tự, kèm gợi ý ref), tham chiếu output_template và tham chiếu policies nếu có.
  3. Tuân thủ model_variants: nếu agent có khai báo cấu hình variant riêng cho công cụ của adapter này (ví dụ model_variants.claude.prompt_append), hãy chèn nó vào; bỏ qua cấu hình variant của các công cụ khác.

Yêu cầu kiểm thử

Mỗi package adapter bắt buộc phải có các tệp kiểm thử trong thư mục src/__tests__/:

Kiểm tra đường dẫn đầu ra chính xác cho một agent mẫu — đảm bảo adapter xuất file đúng vị trí thư mục quy định. Kiểm tra nội dung chứa đầy đủ các trường id, phase, và toàn bộ các bước workflow của agent trong file kết quả. Ít nhất một kiểm thử toMatchSnapshot() trên toàn bộ file kết quả — giúp phát hiện ngay lập tức bất kỳ thay đổi định dạng nào khi review.

Danh sách Adapter đã đăng ký

Adapter name Đường dẫn đầu ra Ghi chú
Universal universal AGENTS.md (index) + .sdlc/agents/<id>.md (mỗi file cho 1 agent) Mức độ cơ bản (baseline) — Markdown thuần túy có thể được đọc bởi bất kỳ công cụ AI nào (Codex, Cursor, Windsurf, Gemini CLI, Zed, ...). Luôn giữ adapter này hoạt động. Không tự định tuyến: người dùng gọi agent bằng tên hoặc mở trực tiếp tệp tin.
Claude Code claude-code .claude/agents/<id>.md (mỗi file cho 1 agent) YAML frontmatter: chứa name, description (tiền tố [SDLC:<phase>] để tự động định tuyến), model (ánh xạ từ model_hint: fast ➔ Haiku, balanced ➔ Sonnet, high-reasoning ➔ Opus), và tools. Áp dụng model_variants.claude.
Copilot copilot .github/copilot-instructions.md (index) + .github/prompts/<id>.prompt.md (mỗi file cho 1 agent) Các tệp prompt sử dụng frontmatter mode: agent kèm theo description tóm tắt trên một dòng. Áp dụng model_variants.copilot.
Ở Phase 2 (theo yêu cầu FR-11 của SA design), chúng ta sẽ thay thế cấu hình hardcode ADAPTERS trong packages/cli/src/commands/build.ts bằng một hệ thống plugin interface (sử dụng adapters: ["@org/sdlc-adapter-x"] trong config). Cho tới lúc đó, các adapter tạm thời nằm chung trong monorepo này.

Quy trình thêm Adapter mới

  1. Khởi tạo thư mục packages/adapters/<tool>/ với src/index.ts export một class hiện thực hóa ToolAdapter.
  2. Đảm bảo đáp ứng đầy đủ các thuộc tính đầu ra và viết đủ các loại kiểm thử nêu trên (path contract + content contract + snapshot).
  3. Đăng ký adapter mới trong đối tượng ADAPTERS tại tệp packages/cli/src/commands/build.ts.
  4. Khai báo thêm tên target vào tài liệu cấu hình sdlc.config.yaml và tệp README.md ở thư mục gốc.
  5. Bổ sung một hàng vào bảng Danh sách Adapter đã đăng ký ở trên, nêu rõ phiên bản công cụ đã được xác thực và các lưu ý về định dạng file.
  6. Nếu target cần hành vi installer riêng, ghi rõ trong tài liệu CLI/wizard thay vì để adapter đọc trạng thái host hoặc tự ghi file.
