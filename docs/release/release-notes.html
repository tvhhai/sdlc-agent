Release Notes — Nhật ký sản phẩm

Trạng thái nền của sản phẩm và nhật ký từng lần sửa / audit / bổ sung — để nắm tiến hóa của dự án mà không cần đọc git log.

Mục đích. Ghi lại sản phẩm gồm những phần nào, làm được gì; và nhật ký thay đổi có ý nghĩa.
Quan hệ doc khác. Mức trung bình ở đây — chi tiết kiến trúc xem SA Design, folder/code xem Codebase Guide, hợp đồng adapter xem Adapter Contract.
Cách đọc. Phần Sản phẩm nền mô tả trạng thái ổn định. Nhật ký cải tiến xếp mới nhất lên đầu.
Cập nhật. MD là nguồn chuẩn (RELEASE_NOTES.md); HTML là bản render. Chèn entry mới vào đầu nhật ký rồi re-render — không sửa HTML tay. Rule đầy đủ trong docs/CLAUDE.md.

Sản phẩm nền (Baseline)

sdlc-agent là một build engine: biến một nguồn định nghĩa agent chuẩn (agents/*.yaml) thành nhiều định dạng cho từng AI tool. Một nguồn sự thật, nhiều target render ra deterministic.

Thành phần

PhầnVai trò
agents/*.yamlNguồn chuẩn. 6 agent phủ vòng đời SDLC: requirement-analyst, planner, solution-architect, coder, test-generator, code-reviewer. Mỗi file = role + workflow + inputs + tham chiếu policy + output template.
policies/*.mdChecklist theo lĩnh vực (security, performance, testing-patterns, debugging-and-recovery, conventions…), agent nạp qua trường policies:. Tái sử dụng giữa các agent.
templates/*.mdKhuôn artifact đầu ra (PRD, plan, ADR, test plan, review report…).
packages/coreZod schema (strict: kebab-case id, semver, enum phase), loader YAML, resolver biến {{var}}, merger config, kiểu dùng chung.
packages/cliLệnh sdlc init / validate / build / list.
packages/adapters/*Mỗi adapter (universal, claude-code, copilot) là một ToolAdapter thuần & deterministic, render output cho một tool.
Output (generated)AGENTS.md + .sdlc/ (universal), .claude/agents/ + .claude/commands/ + .claude/sdlc/ (Claude Code), .github/copilot-instructions.md + .github/prompts/ (Copilot). Commit có chủ đích, không sửa tay.
docs/SA Design, Codebase Guide, Adapter Contract, và tài liệu này.

Chức năng

  • validate — kiểm schema + tính toàn vẹn tham chiếu (mọi output_template/policies phải tồn tại) + tên target hợp lệ.
  • build — render mọi target → ghi file + .sdlc/build-manifest.json; so hash để cảnh báo drift.
  • list — liệt kê agent đã định nghĩa.
  • init — scaffold sdlc.config.yaml.
  • Drift detection — integration test fail nếu output đã commit lệch với output render lại, buộc rebuild + commit cùng nhau.

Định vị hiện tại

Sản phẩm thực tế đang là in-place renderer + canonical catalog (chạy pnpm sdlc build trong chính repo này) — hoạt động đầy đủ. Tầm nhìn installable framework (npx sdlc-agents init cho team khác cài) vẫn là roadmap, chưa ship. Xem entry audit để biết khoảng cách cụ thể.

Vì sao. Bundle self-contained (entry dưới) giúp copy được .claude/, nhưng copy vô nghĩa nếu subagent bên trong không dùng được tool nào. 2026-06-16 Audit ngoài — định vị "in-place vs installable" audit

Bối cảnh. Một AI review độc lập chấm: build engine 7/10, agent content 6.5/10, installer/distribution 2/10; kết luận "chưa phải Agent Skills framework hoàn chỉnh".

Kết quả kiểm chứng (đã verify bằng code). Các claim sự kiện đúng:

  • package.json để private: true, không có dist build, bin trỏ TS source → npx sdlc-agents init chưa chạy được như docs ngụ ý.
  • init chỉ ghi sdlc.config.yaml, không copy agents//policies//templates/ → repo trống không build được.
  • Output không self-contained: .claude/agents/*.md chỉ tham chiếu policies/*.md, adapter không copy nội dung vào bundle → copy riêng .claude/agents/ sẽ gãy link. (Đã xử lý 2026-06-16 — xem entry trên.)
  • Chưa có SKILL.md adapter; Bước 3 dogfood golden case chưa xong.

Phân định. "Không dùng được" chỉ đúng cho bản phân phối (installable); bản in-place renderer dùng được ngay (validate/build/list/test/typecheck/lint đều pass). Khoảng cách là giữa docs/vision và artifact thật, không phải lỗi kiến trúc.

Tác động / hành động. Hai fix nhỏ ROI cao: (1) init bootstrap luôn agent set; (2) bundle self-contained — đã làm 2026-06-16. Sau đó mới tới dist + bỏ private + publish. Trước mắt: chỉnh docs cho khớp thực tế (in-place catalog, pre-pilot).
